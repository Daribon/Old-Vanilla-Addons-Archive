Overview:

This mod provides you with 10 action bars that can use all 120 action buttons. Each bar is hella configurable. You can also place buttons that aren't attached to any bar (referred to as "floaters"). Lastly, the mod also creates control boxes that can be used to setup a neat menu for showing and hiding bars without the need for macros.
Back to Top

The Features:

  • You can define the number of buttons each bar holds.
  • The appearance of the buttons is fully customizable: size, transparency, space between buttons, numbers of rows to use, hide empty buttons, and more.
  • You can configure a background to display behind the buttons.
  • Bars can be set to display when you're in combat, have a hostile target, when you move the mouse over the bar, when you hold down a specific key, and more.
  • You can configure "control boxes", special buttons that can be configured to show a bar when clicked or you move the mouse over them.
  • Bars and control boxes can be attached to other bars and control boxes so that they move together.
  • You can configure "floaters", buttons that are free of any bar.
  • You can define macros that run in response to game events. These macros do not use up any of your base 18 macros.
  • You can configure the pet action, shapeshift, bag, and micro menu bars.
  • You can save and load different settings without having to relog or reload the ui.

    Slash Commands:

  • /dab - toggle the options window
  • /dab barshow # - show bar #
  • /dab barhide # - hide bar #
  • /dab bartoggle # - show bar # if it's hidden else hide it
  • /dab barswap barA barB - swap the buttons on barA with those on barB
  • /dab unswap - swap the last two bars you swapped using the /dab barswap command
  • /dab floatershow # - show floater #
  • /dab floaterhide # - hide floater #
  • /dab floatertoggle # - show floater # if it's hidden else hide it
  • /dab clearbar # - removes the actions from all buttons on bar #
  • /dab setkeybar # - set bar # as the bar the dynamic keybindings use
  • /dab hideallbars - hide all bars
  • /dab showallbars - show all bars

    Options Overview:

    You access the options window by typing /dab.
    The top set of buttons on the left-side of the screen are used to select an action bar to configure.
    The Other Bars button is used to configure the Pet Action, Shapeshift, Bag, and Menu bars.
    The Num Buttons button brings up a window where you can set the number of buttons each bar holds.
    The Floaters button brings up the options window for buttons that aren't attached to any bar.
    On Event Macros is where you define macros to be run in response to game events.
    Misc Options holds various options that don't fit in other categories.
    Clicking the Unlock Dragging button will allow you to move all frames created by DAB by dragging them to where you want them. Clicking the button again will re-lock all the frames. You can also hold Ctrl to override the dragging lock. A bar's label can also be always be used to drag a bar.
    The Lock Buttons button will keep you from dragging actions out of all the buttons.
    The Show Button IDs button will bring up a display of each button's ID in the middle of the button.
    The Quick Save button will save your settings using your character's name as the settings name.
    The Quick Load button will load your settings using your character's name as the settings name.

    All edit boxes require you to hit enter before they update the value they modify.
    All sliders have an edit box attached to them where you can type in a value instead of using the slider. For most of those edit boxes you can type in a value that's greater or less than what the slider allows. This will disable the slider until you type in a value that's within the slider's range.

    Bar Options:

  • Hide: This hides the bar.
  • Hide Empty Buttons: This hides buttons on the bar that don't hold an action.
  • Collapse Empty Buttons: This hides empty buttons and causes other buttons to shift to the left to fill in the blank spots left by the empty buttons.
  • Hide When an Action Button is Pressed: This will cause the bar to be hidden right after you click any of its buttons.
  • Show When In Combat: The bar will be shown when you enter combat, whether through entering attack mode or you gain aggro. It will be hidden when you leave combat.
  • Show When Out of Combat: The bar will be shown when you leave combat and will hide when you enter combat.
  • Show For Friendly Target: The bar will be shown when you target a friendly unit. It will hide if you target a hostile unit or if you don't have a target.
  • Show For Hostile Target: The bar will be shown when you target a hostile unit. It will hide if you target a friendly unit or if you don't have a target.
  • Show When Mouse Moves Over the Bar: The bar will be shown when you move the mouse over the area of the screen it occupies. It will be hidden when you move the mouse off the bar.
  • Show While A Key Is Pressed: The bar will shown while you hold a key you set in the keybindings menu and will hide when you release the key.
  • Right-click Bar: This sets a bar to use actions from when you right-click a button. The action used will match the location of button clicked. For example, if you set right-click bar to 3 and right-click the 4th button on the bar, the 4th button on bar 3 is what will actually be pressed.
  • Middle-click Bar: This sets a bar to use actions from when you middle-click a button.
  • Default Target: Type in a unit id to use as the bar's default target. That unit will be the one targetted whenever you cast a spell on the bar if you don't have a target selected. Valid unit id's can be found at http://www.wowwiki.com/Units.
  • Button Size: This sets the height and width of buttons on the bar in pixels. It is affected by your UI scale.
  • Button Transparency: This sets the alpha level of buttons on the bar.
  • Horizontal Spacing: This sets the amount of space between buttons in a row.
  • Vertical Spacing: This sets the amount of space between rows.
  • Button Rows: This sets the number of rows in which the buttons display. Setting the rows equal to the number of buttons on the bar will give you a 1 column vertical bar.
  • Attach to Frame: Select another bar or control box to attach this bar to. This bar will drag along with whatever frame you attach it to.
  • Attach Point: Select which part of the bar attaches to the frame selected in the above option.
  • Attach To: Select which part of the frame the bar attaches to.
  • X Offset: Set the horizontal distance between the bar and the frame it's attached to.
  • Y Offset: Set the vertical distance between the bar and the frame it's attached to.
  • Use For Shapeshift Form: Select a shapeshift form this bar is used for.
  • When I Shapeshift: Select what happens when you shapeshift into the bar's selected form.
  • Copy Settings From Bar: Select another bar to copy settings from then click the copy button.

    Background Options:

  • Background Style: Use this to select a different style for the background. The main difference between the styles is the border art. Style 1 is the only style where you can define a custom background texture. It's also the only style that can be fully opaque. The background color for style 3 can't be changed.
  • Background Color: The name says what it is.
  • Background Texture: Use this box to type in the filename without a file extension of a custom texture to use for the background. Texture files must be placed in your DiscordActionBars folder. Custom textures only work with background style 1.
  • Background Alpha: Use this to set the transparency of the background.
  • Padding: Use this to set the amount of space between the edges of the background and the bar's buttons.
  • Border Color: Guess.
  • Border Texture: Use this box to type in the filename without a file extension of a custom texture to use for the background's border. Texture files must be placed in your DiscordActionBars folder. Custom textures only work with background style 1.
  • Border Alpha: Use this to set the transparency of the background's border.
  • Button Border Color: Use this to set the color of the border that surrounds the action buttons in the bar.
  • Button Border Alpha: Use this to set the transparency of the border that surrounds the action buttons in the bar.
  • Button Border Padding: Use this to set the amount of space between a button and its border.
  • Copy Settings From Background of Bar: Choose a bar and click the copy button to copy background settings from that bar to the bar you're editing.

    Control Box Options:

  • Text: Used to set the text displayed in the middle of the control box.
  • Text Color: Used to set the text's color.
  • Background Color: Used to set the background's color.
  • Border Color: Used to set the border's color.
  • Hide: Check this to hide the control box.
  • Show Bar On Click: Checking this will cause the control box's bar to be shown if it's hidden or hidden if it's visible whenever you click the control box.
  • Hide Other Bars On Click: Checking this will hide all other bars set to be shown when thier control box is clicked whenever you click this control box.
  • Show Bar On Mouseover: Checking this will cause the control box's bar to be shown when you move the mouse over the control box. The bar will hide after a short delay when you move the mouse off the control box (gives you time to move the mouse from the box to the bar). The length of the delay can be adjusted in Misc Options. The bar will hide immediately when you move the mouse off of it.
  • Mouseover Background Color: Check the checkbox to cause the control box's background to change color when you move the mouse over it. Use the color box to set the color the background changes to.
  • Mouseover Text Color: Check the checkbox to cause the control box's text to change color when you move the mouse over it. Use the color box to set the color the text changes to.
  • Show Top/Left/Right/Bottom Border: Uncheck the boxes for specific borders you want to hide. You can use this to setup neat menus with one continous border around multiple control boxes.
  • Perform Macro On Click: Type in the name of a macro to perform when the control box is clicked. This can be used to expand what happens when the box is clicked beyond just toggling a single bar.
  • Height: Sets the height of the control box.
  • Width: Sets the width of the control box.
  • Text Alpha: Sets the transparency of the control box's text.
  • Text Size: Sets the font size of the control box's text.
  • Background Alpha: Sets the transparency of the control box's background.
  • BorderAlpha: Sets the transparency of the control box's border.
  • Attach to Frame: Select another bar or control box to attach this control box to. This control box will drag along with whatever frame you attach it to.
  • Attach Point: Select which part of the control box attaches to the frame selected in the above option.
  • Attach To: Select which part of the frame the control box attaches to.
  • X Offset: Set the horizontal distance between the control box and the frame it's attached to.
  • Y Offset: Set the vertical distance between the control box and the frame it's attached to.
  • Copy Settings From Control Box: Choose a bar and click the copy button to copy settings from that bar's control box to the control box you're editing.

    Num Buttons:

    Free Buttons at the top displays how many buttons you have that aren't attached to any bar and aren't declared as a floater. This box cannot be edited by hand.
    Use the + button next to a bar to add a button to that bar. A new button can potentially show in any location on a bar - they're ordered by thier ID numbers.
    Use the - button next to a bar to remove a button from that bar. The button removed will always be the last button on the bar.

    Floaters:

    Floaters are usable buttons that aren't attached to any bar.
    Create a new floater by clicking the Convert Free Button To Floater button at the top of the options window. You must have a free button - a button not attached to a bar and not already declared as a floater - available to convert. The new floater will appear in the middle of your screen. Make sure you create a floater before you try to start editing one.
    Use the < and > buttons to scroll through your floaters to choose one to edit. The ID of the selected floater appears between those 2 buttons.
    Use the Delete This Floater button to return the floater to the stack of free buttons.
  • Hide: Hides this floater.
  • Hide When Clicked: The button is hidden immediately after it's clicked.
  • Show While A Key Is Pressed: The button is shown only while you hold down a key you set in the keybindings menu and hides when you release that key.
  • Show For Friendly Target: The button is shown only when you have a friendly unit targetted.
  • Show For Hostile Target: The button is shown only when you have a hostile unit targetted.
  • Show When In Combat: The button is shown only when you're in combat.
  • Show When Not In Combat: The button is shown only when you're not in combat.
  • Border Color: This sets the color of the border around the button.
  • Default Target: Type in a unit id to use as the button's default target. That unit will be the one targetted whenever you click that button if you don't have a target selected. Valid unit id's can be found at http://www.wowwiki.com/Units.
  • Middle-click Button: Type in the ID of a button to be used instead of this button when you middle-click on this button. It can be any button, not just another floater.
  • Right-click Button: Type in the ID of a button to be used instead of this button when you right-click on this button. It can be any button, not just another floater.
  • Show For Shapeshift Form: The button only displays while you're in the selected shapeshift form.
  • Button Size: This sets the height and width of the button in pixels.
  • Button Transparency: This sets the alpha level of the button.
  • Button Border Padding: This sets the distance between the edges of the button and it's border.
  • Button Border Alpha: This sets the transparency of the button's border.
  • Attach to Frame: Select a frame from the menu to attach this button to. This button will be dragged whenever you drag the frame it's attached to.
  • Attach Point: Select the part of the button that attaches to the frame selected above.
  • Attach To: Select the part of the frame that the button attaches to.
  • X Offset: Sets the horizontal distance between the button and the frame it's attached to.
  • Y Offset: Sets the vertical distance between the button and the frame it's attached to.

    On Event Macros:

    Here's where you can write your own blocks of code to execute in response to game events using all the normal macro conventions. Choose the event from the dropdown menu at the top of the window and enter the macro's code in the large edit box beneath it. These macros do not use any of your base 18 macros and have a limit of 7500 characters, so go nuts. The macro will be executed every single time the event is thrown. See the section Useful Functions for a list of handy functions this mod provides and see the section Event Macro Examples for examples of how to display a button when your health's low, when your opponent dodges, etc.

    Misc Options:

  • Auto-attack When Using an Action: Checking this will cause you to automatically enter attack mode whenever you use an action. It will check if you're already attacking and if you have a valid target before attempting to toggle auto-attack.
  • Disable Action Button Tooltips: Checking this will stop tooltips from displaying when you move the mouse of an action button.
  • Hide Keybindings Text: Checking this will hide the regular keybindings text on all action buttons.
  • Hide Dynamic Keybindings Text: Checking this will hide the dynamic keybindings text on all action buttons.
  • Show a Text Display of Cooldown Time: Checking this will cause a large number to display in the middle of each button that counts down the amount of time remaining on that button's cooldown.
  • Out Of Range Color: Use this to set the color a button becomes when your target is out of range.
  • Not Enough Mana Color: Use this to set the color a button becomes when you don't have enough mana to use its action.
  • Unusable Color: Use this to set the color a button becomes when its action is unusable.
  • Main Bar for Shapeshifting: Use this to set the bar other bars swap with when you shapeshift.
  • Ghost Bar: Use this to choose a bar to use as the ghost bar. This bar will appear underneath the mouse when you hold down the key you bound to toggle the ghost bar.
  • Options Window Scale: Use this to set the size of the options window.
  • Time to Hide on Mouseout: Use this to set the number of seconds until a bar hides when you move the mouse off its control box if the control box is set to show the bar on mouseover.
  • Cooldown Count Threshold: Use this to set a minimum number of seconds a cooldown has to have for it to be displayed.
  • Load Default Settings: Click this button to return all settings for the mod to default settings.
  • Load Custom Settings: Click this button to load settings from the DAB_Custom.lua file.
  • Save Settings: Type in a name to save your settings as then click the Save button to save them.
  • Load Settings: Select settings in the dropdown menu then click the load button to load them or the delete button to delete them.

    Setting Up Bars That Switch When You Shapeshift:

    First, go to Misc Options and set the Main Bar for Shapeshift to whatever bar number you want the shapeshift bars to switch with. Then go to the options for another bar and set the Use For Shapeshift Form option to whatever form you want to use with that bar. Underneath there, set the When I Shapeshift option to Swap With Main Bar. Once that's done, the next time you actually change stances, stealth, or shift into an animal form the bars will swap. It's important to remember that the correct bar will not be in the Main Bar until you actually change forms. For warriors, this means you need to switch to a different stance from your current one. For druids and rogues, the main bar won't have the correct bar until you stealth or shift to an animal form from humanoid form (so if you setup your bar while stealthed or in an animal form, you won't have the correct bar in the main bar until you leave stealth/animal form and re-enter stealth/an animal form). Another tip for warriors is to remove all the buttons from your main bar before you change stances for the first time or you'll never see those buttons again. Bars only swap buttons and not settings so your shapeshift bars will appear however you set the main bar to appear and your keybindings for the main bar will be applied to whatever shapeshift bar it currently holds.

    Useful Functions:

    barnum is a number from 1 to 10 specifying one of your action bars.
    floaternum is the button ID of a floater.
  • DAB_Bar_Hide(barnum): Hides the specified bar.
  • DAB_Bar_Show(barnum): Shows the specified bar.
  • DAB_Bar_Swap(barnum1, barnum2): Switches the buttons on barnum1 with those on barnum2.
  • DAB_Bar_Toggle(barnum): Hides the specified bar if it's visible, shows it if it's hidden.
  • DAB_Floater_Hide(floaternum): Hides the specified floater.
  • DAB_Floater_Show(floaternum): Shows the specified floater.
  • DAB_Floater_ShowWhenUsable(floaternum): This will cause DAB to wait to show the floater until its action becomes usable and isn't cooling down.
  • DAB_Floater_Toggle(floaternum): Hides the specified floater if it's visible, shows it if it's hidden.
  • DAB_Floater_TimeToHide(floaternum, seconds): Allows you to specify a number of seconds to wait before the specified floater gets hidden. Useful for buttons set to show when your target dodges, etc.
  • DAB_AttackTarget(): Calls AttackTarget() only if you have an attackable target and you're not currently in attack mode.
  • DAB_Run_Macro(macroname): Executes the macro with the specified macro name. Yes, it works within a macro to create looping macros.
  • DAB_SetKeybindingBar(barnum): Applies the dynamic keybindings to the specified bar.
  • DAB_Get_CurrentForm(): Returns the number and name of your current shapeshift form - syntax: number, name = DAB_Get_CurrentForm(). Returns 0 and Humanoid if you're not shapeshifted.
  • DAB_Check_Mouseover(frame): This function returns true if the mouse is over the specified frame. Frame is the actual frame object, not just the frame's name. Works with any frame, not just DAB's.
  • DAB_Options_LoadSettings(settingsname): Loads the specified settings.
  • DAB_IsInContext(friendlytarget, hostiletarget, combat, notincombat, ontrue): For the parameters pass true if the bar or floater is supposed to show for that parameter or false if it isn't. The function will return true if the bar or floater is supposed to be shown. If the onetrue parameter is true then only one of the conditions has to be true for the function to return true.

    Event Macro Examples:

    Showing a floater when your health drops below a certain percentage:
    Choose the Unit's Health Changed event and enter the following for the macro text. We'll say the percent you want is 50% so it'll be shown as .50 below and the floater's number is 23.
    if arg1=="player" then if UnitHealth("player")/UnitHealthMax("player") < .50 then DAB_Floater_Show(23) else DAB_Floater_Hide(23) end end
    This will show floater number 23 when you health is below 50% and hide it if your health is 50% or greater. So it'll hide the button if you get a heal that takes you above 50% before you press the button.

    Showing a floater when your target dodges:
    Choose the 'You Miss' Chat Message event and enter the following for macro text. We'll say the floater is number 23.
    if string.find(arg1, "dodges") then DAB_Floater_Show(23); DAB_Floater_TimeToHide(23, 5); end
    This will cause floater number 23 to show up whenever your target dodges your attack and it will automatically hide 5 seconds later.

    Swapping bars when your target enters melee range:
    Choose OnUpdate from the event macros dropdown then the enter the code below for your macro.
    if CheckInteractDistance("target", 1) then if (not meleebar) then setglobal("meleebar", 1); DAB_Bar_Swap(bar1, bar2); end elseif meleebar then DAB_Bar_Swap(bar1, bar2); meleebar = nil; end
    Replace bar1 and bar2 with the numbers of the 2 bars you want to swap. This script will swap the 2 bars when the target moves within 5 yards of you then swap them again when the target moves further away than that.
    If you want to automatically swap back to your ranged bar when you lose your target, select You Change Targets from the events dropdown menu and enter the following for the macro.
    if (not UnitName("target")) and meleebar then DAB_Bar_Swap(bar1, bar2); meleebar = nil; end

    Known Issues:

    Do not cross-attach frames. If bar 1 is attached to bar 3, do not also attach bar 3 to bar 1. WoW doesn't like that and will crash.
    Using /say, /party, /raid, and /whisper in an On Event Macro will cause you to get disconnected.
    If your Pet bar or Shapeshift bar are disappearing when you click a button, disable CT_BarMod.
